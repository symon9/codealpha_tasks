## 📌 Task Brief

> Pick a popular app/website (e.g., Zomato, Flipkart, Paytm). Analyze its user journey, strengths, and weaknesses. Suggest 3 improvements with sample sketches or mockups. Submit as a case study report (4–5 pages).

---

## 🎯 App Analyzed: Duolingo

**Duolingo** is one of the world's most popular language-learning applications, used by hundreds of millions of people globally. It leverages gamification streaks, points, and rewards to encourage daily learning habits.

The case study examines Duolingo's UX through the lens of a target user, mapping their journey, identifying friction points, and proposing evidence-based improvements backed by mockup designs.

---

## 📄 Case Study Summary

### User Persona

**Leo** a 22-year-old student learning French for travel, seeking to build a consistent daily habit.

### User Journey Mapped

1. App Download & Onboarding
2. Language & Goal Selection
3. Starting a Lesson
4. Completing or Failing a Lesson
5. Return to Home Screen

---

### ✅ UX Strengths Identified

| Strength                       | Why It Works                               |
| ------------------------------ | ------------------------------------------ |
| Gamification (streaks, points) | Drives daily retention and habit formation |
| Simple, colorful interface     | Lowers barrier to entry for beginners      |
| Short lesson format            | Fits easily into daily routines            |
| Immediate in-lesson feedback   | Reduces uncertainty during exercises       |

---

### ⚠️ UX Weaknesses Identified

| Weakness                            | Impact on User                                  |
| ----------------------------------- | ----------------------------------------------- |
| No detailed error explanation       | Users repeat mistakes without understanding why |
| Poor loading visibility             | Users think the app has frozen                  |
| Accessibility gaps (color contrast) | Excludes color-blind users                      |
| Limited learning path flexibility   | Advanced users feel constrained                 |
| Audio overlap issues                | Confuses users during exercises                 |

---

### 💡 3 Improvement Proposals

**Improvement 1 Lesson Review Screen**
Add a post-lesson screen showing incorrect answers alongside correct answers and short explanations, helping users learn from mistakes rather than just retrying blindly.

**Improvement 2 Enhanced Loading Screen**
Redesign the loading experience with a visible progress bar and animated Duolingo mascot, communicating that the app is working and reducing perceived wait time.

**Improvement 3 Accessibility Settings Panel**
Introduce a dedicated accessibility settings screen with a Colorblind Mode toggle, visual symbols (✔ ✖) alongside color indicators, and granular audio control settings.
